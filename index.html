Visión General

Este dashboard le permite navegar y comparar los términos y condiciones de los principales modelos de lenguaje disponibles en el mercado. La información ha sido extraída y simplificada para una consulta rápida. Siempre consulte los documentos originales para obtener detalles completos.

Principales hallazgos generales

  • Privacidad de Datos: OpenAI y Google tienen enfoques variables. Los planes empresariales y APIs suelen ofrecer más control, con opciones para excluir datos del entrenamiento. NotebookLM destaca por no usar datos para entrenar modelos.
  • Propiedad del Contenido: Generalmente, el usuario es propietario del contenido generado, pero las plataformas se reservan derechos para mejorar servicios y cumplir políticas.
  • Retención de Datos: Todos los servicios retienen datos por ciertos periodos (ej. 30-55 días) por seguridad o cumplimiento, incluso si se opta por no entrenar modelos con ellos.
  • Costos: Varían desde gratuitos con limitaciones, hasta planes de suscripción mensual y modelos de pago por uso (tokens) para APIs.

Privacidad de Datos - Visión General

La privacidad de los datos es un aspecto crucial. Aquí se resumen los puntos más importantes.

Aspectos clave de privacidad

  • OpenAI (planes básicos): Recopila datos de interacciones para entrenamiento, con opción de exclusión.
  • OpenAI Enterprise/API: Mayor control de datos, no se usan para entrenamiento por defecto.
  • Google Gemini App/Advanced: Google puede usar contenido para mejorar servicios; la API permite marcar datos como "no utilizables para entrenamiento".
  • NotebookLM: Destacado No utiliza datos de usuario para entrenar modelos de IA.

Definición de Información Personal

PlataformaDefinición de Información Personal
OpenAICualquier dato que pueda identificar a una persona (nombres, direcciones, emails, etc.).
Google (General)Información proporcionada a Google o creada al usar sus servicios que identifica personalmente al usuario (ej. nombre, email, o datos vinculables a dicha información).
NotebookLMSigue la política general de Google. Datos que permiten identificar directa o indirectamente.

Recopilación de Datos

Plataforma/Servicio Tipos de datos recopilados Finalidad principal
OpenAI ChatGPT (Free/Plus/Pro) Datos de cuenta, prompts, respuestas, datos de interacción. Mejora del servicio, entrenamiento de modelos (con opción de exclusión), cumplimiento.
OpenAI ChatGPT (Team/Enterprise/API) Datos de cuenta, prompts, respuestas (contenido de API no usado para entrenamiento por defecto), metadatos. Operación del servicio, seguridad. Los datos de clientes API y Enterprise no se usan para entrenar modelos de OpenAI por defecto.
Google Gemini App/Advanced Contenido creado/subido/recibido (prompts, respuestas), información personal de la cuenta Google, datos de uso. Operar servicios, mejorar productos, personalizar, seguridad. Puede usarse para entrenar modelos.
Google Gemini API Input, output, datos técnicos. El usuario puede marcar input/output como "no utilizable para entrenamiento". Operar y mejorar API, seguridad.
Google NotebookLM Documentos subidos, consultas, respuestas generadas, datos de uso, diagnóstico, metadatos de cuenta Google. Operar el servicio, personalizar respuestas. No para entrenar modelos de IA.

Retención de Datos

Plataforma/Servicio Período de retención principal Justificación
OpenAI ChatGPT (consumidor) Conversaciones retenidas para entrenamiento (a menos que se deshabilite el historial). Datos retenidos 30 días por seguridad incluso con historial desactivado. Mejora del servicio, seguridad.
OpenAI API / Enterprise Por defecto, no se retienen datos para entrenamiento. Pueden retenerse hasta 30 días para revisión de abuso/mal uso. Seguridad, cumplimiento de políticas.
Google Gemini API Input/Output pueden ser retenidos hasta por 55 días para supervisión de abusos (si no se opta por exclusión). Datos de Búsqueda Fundamentada: 30 días. Seguridad, depuración.
Google NotebookLM Indefinido, mientras la cuenta esté activa o hasta que el usuario elimine los datos. Permitir al usuario acceder y usar sus notebooks.

Nota Importante: Los periodos de retención pueden variar y estar sujetos a políticas específicas y requisitos legales. Siempre es recomendable revisar los términos más actuales de cada servicio.

Compartición de Datos con Terceros

Plataforma ¿Comparte con terceros? Principales condiciones.
OpenAI Puede compartir con proveedores de servicios, afiliados, por razones legales o con consentimiento. Los datos de clientes API y Enterprise no se comparten para entrenar modelos de terceros.
Google (General) Con consentimiento, con administradores de dominio, para procesamiento externo (proveedores bajo instrucciones), o por razones legales.
Google NotebookLM Sigue la política general de Google. Puede compartir con proveedores de servicios para operar (ej. alojamiento), por requisitos legales, para proteger seguridad, o con consentimiento.

Uso de Datos para Entrenamiento de Modelos

Plataforma/Servicio ¿Usa datos para entrenar modelos? Opciones de exclusión (Opt-out)
OpenAI ChatGPT (Free/Plus/Pro) Sí, las conversaciones se usan por defecto. Sí, se puede desactivar el uso de datos para entrenamiento en la configuración (desactivar historial de chat).
OpenAI API / Enterprise No por defecto. OpenAI no utiliza los datos enviados a través de la API ni los datos de clientes de ChatGPT Team/Enterprise para entrenar sus modelos. N/A (ya es por exclusión por defecto).
Google Gemini App/Advanced Sí, Google puede usar el contenido para mejorar sus servicios, incluyendo tecnologías de aprendizaje automático (a menos que se indique lo contrario para servicios específicos o mediante controles de actividad). Depende de los controles de actividad de la cuenta de Google. Para algunas interacciones, la revisión humana puede estar activada por defecto con opción a desactivarla.
Google Gemini API (en AI Studio) Por defecto, Google puede usar input/output para mejorar modelos. Sí. El usuario puede marcar su input/output como "no utilizable para entrenamiento" a través de la configuración de la API.
Google NotebookLM No. No usa datos recopilados del usuario (documentos, consultas, respuestas) para entrenar modelos de IA. N/A (no los usa para entrenamiento).

Recomendaciones para el control de datos de entrenamiento:

  • Para OpenAI consumidor: Desactiva el historial de chat si no deseas que tus conversaciones se usen para entrenamiento.
  • Para APIs (OpenAI y Gemini): Utiliza las opciones de exclusión si están disponibles y son necesarias para tu caso de uso.
  • Para máxima confidencialidad del contenido: Considera NotebookLM o planes Enterprise/API con políticas claras de no-entrenamiento.

Propiedad del Contenido Generado

Plataforma/Servicio Propiedad del Contenido Generado (Output) Derechos de la Plataforma sobre el Contenido
OpenAI (General ChatGPT) El usuario es propietario del output. OpenAI puede usar el contenido (input y output) para proveer y mantener los servicios, cumplir con la ley y sus políticas. Para mejorar modelos si el usuario no opta por la exclusión.
OpenAI API / Enterprise El usuario es propietario del output. OpenAI no usa el contenido de la API/Enterprise para mejorar sus modelos por defecto. Lo usa para proveer y mantener servicios, cumplir ley y políticas.
Google Gemini App/Advanced El usuario retiene los derechos de propiedad intelectual que ya tenga sobre el contenido que envía. Google no reclama propiedad sobre el output que el usuario genera. Google obtiene una licencia para usar el contenido (input y output) para operar, proveer y mejorar los servicios, y desarrollar nuevas tecnologías.
Google Gemini API El usuario es propietario del output. Google no reclama propiedad sobre el output. Google puede usar el contenido para operar y mejorar los servicios de API (a menos que se excluya del entrenamiento).
Google NotebookLM NotebookLM no reclama la propiedad del contenido creado. Las respuestas se basan en las fuentes del usuario. Google puede usar los datos para operar y mejorar NotebookLM, pero no para entrenar modelos de IA generales.

Punto Clave: Aunque generalmente eres dueño de lo que creas, las plataformas se reservan ciertos derechos para operar y mejorar sus servicios. Es crucial entender estos matices, especialmente para uso comercial.

Límites de Uso y Restricciones

Todas las plataformas imponen límites para asegurar la disponibilidad y prevenir abusos. Estos pueden ser sobre la cantidad de solicitudes, tokens por minuto, etc.

Plataforma/Servicio Tipos de Límites Comunes
OpenAI ChatGPT (Free) Límites de uso, acceso restringido en horas pico, límites en la longitud de mensajes.
OpenAI ChatGPT (Plus/Team) Límites más altos que Free, pero aún existen (ej. GPT-4 tiene límites de mensajes cada ciertas horas). Límites de tokens.
OpenAI API Límites de tasa (rate limits) por tokens por minuto (TPM), solicitudes por minuto (RPM), solicitudes por día (RPD). Varían según el modelo y el nivel de uso (tier).
Google Gemini App/Advanced Límites de frecuencia de uso no especificados detalladamente para la app, pero existen para asegurar la calidad del servicio. Gemini Advanced (con Google One) puede tener límites más altos.
Google Gemini API (AI Studio) Nivel gratuito: Límites más bajos (ej. 60 RPM).
Nivel de pago: Límites más altos y escalables (ej. hasta 2000 RPM), RPD ilimitado en algunos casos. Medidos en RPM, RPD, TPM.
Google NotebookLM Versión gratuita: Hasta 100 cuadernos, 50 fuentes/cuaderno, 500k palabras/fuente, 50 consultas/día.
Versión Plus: Límites ampliados (ej. 500 cuadernos, 300 fuentes/cuaderno, 500 consultas/día).

Importante: Estos límites pueden cambiar. Consulta siempre la documentación oficial para los valores más recientes y específicos de tu plan o nivel de API.

Medidas de Seguridad

Las plataformas implementan diversas medidas para proteger los datos de los usuarios.

Plataforma Medidas de Seguridad Destacadas
OpenAI Cifrado en tránsito (HTTPS). Para Enterprise/API: cifrado en reposo (AES-256), SOC 2 Tipo 2 compliance, controles de acceso, revisión de seguridad de datos.
Google (General / Gemini / NotebookLM) Cifrado en tránsito y en reposo, infraestructura global segura, revisión de prácticas de recopilación/almacenamiento/procesamiento, controles de acceso físico y lógico, verificación en dos pasos para cuentas. Cumplimiento de estándares como ISO 27001, SOC 2/3.

Prácticas Recomendadas por el Usuario:

  • Utilizar contraseñas fuertes y únicas.
  • Activar la autenticación de dos factores (2FA) siempre que sea posible.
  • Ser cauteloso con la información sensible que se introduce en los modelos.
  • Revisar y gestionar los controles de actividad y privacidad de la cuenta.
  • No compartir credenciales de API.

Recomendaciones Generales

Al utilizar modelos de lenguaje, considera los siguientes puntos para una experiencia más segura y alineada con tus necesidades:

Para Proteger tu Privacidad:

  • Revisa las Políticas: Antes de usar un servicio, especialmente con datos sensibles, lee sus términos y políticas de privacidad.
  • Minimiza Datos Sensibles: Evita introducir información personal identificable (PII), datos financieros, o secretos comerciales si no es estrictamente necesario y si la plataforma no ofrece las garantías adecuadas (ej. planes empresariales con BAA).
  • Usa Opciones de Exclusión: Si la plataforma lo permite (ej. APIs de OpenAI/Gemini, configuración de ChatGPT), opta por excluir tus datos del entrenamiento de modelos si la confidencialidad es primordial.
  • Considera la Retención: Sé consciente de por cuánto tiempo la plataforma podría retener tus datos.
  • Servicios Específicos: Para trabajar con tus propios documentos de forma confidencial, Google NotebookLM es una buena opción ya que no usa tus datos para entrenar modelos de IA.

Para la Propiedad Intelectual:

  • Verifica la Propiedad: Aunque generalmente eres dueño del output, entiende los derechos que la plataforma se reserva.
  • Uso Comercial: Si planeas usar el contenido generado comercialmente, asegúrate de que los términos de la plataforma lo permitan y que tienes los derechos necesarios sobre cualquier input que hayas proporcionado.

Para la Seguridad:

  • Credenciales Seguras: Protege tus claves de API y credenciales de cuenta. No las compartas ni las incrustes directamente en código del lado del cliente.
  • Monitorea el Uso: Especialmente con APIs de pago por uso, monitorea tu consumo para evitar sorpresas.

Disclaimer: Este dashboard es una guía y simplificación. Las políticas y términos de los servicios pueden cambiar. Siempre consulta los documentos oficiales y actualizados de cada proveedor. Para asesoramiento legal específico, consulta a un profesional.
